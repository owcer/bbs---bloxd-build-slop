remember, the environment has a watchdog timer. async and promises don't exist, and setTimeout and its cousins don't exist either. also, the world code init itself is very prone to getting interrupted, so run minimal code there. use sbr calls and sbr.ior and sbr.ior.levels for pretty much everything, as they are safe from interruptions. reprint the code for the library when providing the code for the request.

your code should look something like this:
```js
[code for library]
[declaration of a functions that uses sbr or sbr.ior, where sbr is generally used to large scale rects and sbr.ior is for fine grain control (like random texture)]
[an array of arrays the these mini functions]
[declaration of manager functions for sub-sections of the build calling ior on their respective set of mini functions once]
[an array of the manger functions]
[declaration of a single master function for each big section of the build calling ior on the array of manager functions]
[a single sbr.ior.levels call on the master function]
```
because remember, the code you write cannot take more than about, 10ms, and functions cannot simply do multiple multiple-second tasks synchronously. however, you can always call sbr.ior on an array of callbacks, and it will execute them asynchronously, and the function will not be interrupted.
